# Evan 在 DJI 的舆情系统工作总结（海外平台）

## 核心观点

DJI 舆情系统最突出的设计是**多维度结构化标注**和**人机分工明确**。系统自动生成基础字段，人工负责需要判断的标注维度。这套标注体系是 AI 替代性最高的环节——因为它已经被高度结构化。

## 数据采集三层架构

| 采集方式 | 机制 | 优点 | 缺点 |
|---------|------|------|------|
| **脚本自动抓取** | 关键词+无头浏览器，取前~60条 | 自动化 | 量偏少、准确度低、无法判断真实性 |
| **供应商数据(KOC)** | 合作方提供 | 稳定、品牌传播视角 | 偏向品牌正面，真实用户评价有限 |
| **人工账号监测** | 手动账号+关键词搜索前20条 | 灵活可调、弥补脚本盲区 | 人力成本高 |

## 系统字段结构（DJI 标注体系的核心）

### 系统自动生成字段
- 唯一识别码、业务线代码、原始链接、平台、日期
- 转载追踪、Top排名标识、KOL 状态标识

### 人工判断字段（核心标注环节）
- **内容分类**（单选）：评价 / 视频 / 问答 / Others
- **多维度情感判断**（并行）：
  - DJI 品牌：正 / 负 / 无关
  - 产品1/2/3：正 / 负 / 无关
  - 竞品标注：是否提及 + 情感倾向
- **Classification 自动生成**：系统根据人工标注自动计算等级
- **补充信息**：涉及产品型号、真实性判断(Y/N)、摘要

## 处置工作流（WOM Action）

```
系统生成Class等级 → 选择处理策略 → 如需介入 →
策略制定 + 文案撰写投放 → WOM Action记录 →
Processing → Monitoring(5天观察) → Close(闭环)
```

**状态流转规则**：
- Processing → Monitoring：处置达到KPI要求
- Monitoring → Close：观察5天无反弹
- Close 后：KPI 已达标，无需进一步干预

## 关键引用

> "系统会根据人工标注的判断结果，自动生成对应的 Classification 等级"

> "该阶段是系统中最为KPI考核的关键环节，口碑人员的判断结果将直接影响该舆情被系统如何'定性'"

## 个人评注

DJI 系统最值得借鉴的设计是**将人工判断高度结构化为选择题**——内容分类是单选、情感是三维并行单选、真实性是二选一、问题分类是两级级联选择。这种设计让标注一致性高、培训成本低，也恰好是 AI 最容易达到高准确率的标注形式。

从信息论角度，DJI 的标注体系本质上是一个**编码器**：将高熵的非结构化文本压缩为低熵的结构化标签。这个编码过程的可规则化程度越高，AI 替代的可行性就越大。

与 TEMU 体系的对比：TEMU 强在分流决策流程（ignore/process/observe）和高危响应机制，DJI 强在标注维度的丰富性和结构化程度。两套体系的融合可以取长补短。
