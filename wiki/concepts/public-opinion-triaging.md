# 舆情分流判断

## 概述

分流判断是舆情分析的最后一步决策：决定这条舆情**要不要处理、怎么处理**。来自 TEMU 指挥部的 ignoring/processing/observing 三分法 + DJI 的忽略/干预/升级三分法。

## 四分类定义

| 决策 | 适用条件 | 下游动作 |
|------|---------|---------|
| **立即处理** | P0 / 高影响力 P1 | 拉群→通报→建联→盯进展→闭环归档 |
| **持续观察** | P1(低影响) / P2 / 不确定中高风险 | 定期回查→恶化升级→平息降级 |
| **可忽略** | P3 / 已删除 / 明显虚假且无传播 | 记录归档（保留用于趋势分析） |
| **正面可利用** | 正面+优质 | 转PR评估→合作/转发/归档 |

## 决策流程

```
严重度评级 → P0→立即处理
            → P1→影响力高?→是→立即处理
            → P1→影响力低?→是→持续观察
            → P2/P3→正面?→是→正面可利用
            → P2/P3→负面→持续观察/可忽略
```

## 关键边界规则

### 1. 忽略 ≠ 丢弃
被标记为"可忽略"的内容仍需归档。同类聚集（≥20条/周）→ 升级。

### 2. "观察"的升级/降级
- 观察→升级：同类报告≥3 / 互动增速显著 / 被媒体转载 / 出现变体
- 观察→降级：48-72h互动趋平 / 无新增同类
- 观察→闭环：5天无反弹→Close

### 3. 真实性对分流的影响
- 大概率虚假 + 零传播 → 可忽略（辅助依据）
- 大概率虚假 + 有传播 → **不能降级**
- 参见 [[cases/case-006|案例006]]

### 4. 竞品相关内容
- 公正评测→持续观察（不是负面，但关注评论区）
- 恶意对比→按严重度正常分流
- 竞品攻击我方→至少"持续观察"

## 信息论视角

分流判断是**信息压缩的最终决策节点**：非结构化文本→分类→情感→评级→[处理/观察/忽略]。将 P3 分流为"忽略"是系统的自我保护——不加区分地处理所有信号会导致信道超容。
