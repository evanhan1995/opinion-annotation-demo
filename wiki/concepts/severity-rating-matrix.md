# 严重度评级矩阵（P0-P3）

## 概述

严重度评级是舆情分流判断的**核心决策依据**。评级由三个维度叠加决定：内容严重程度 × 传播影响力 × 发展态势。来自 TEMU 的 KPI 体系（P0/P1 分级）和 DJI 的 Classification 自动生成逻辑。

## 三维叠加模型

```
严重度 = 内容严重程度 × 传播影响力 × 发展态势修正
```

### 基础评级矩阵

| | 低影响力 | 中影响力 | 高影响力 |
|---|---|---|---|
| **高危内容** | P1 | P0 | P0 |
| **中危内容** | P2 | P1 | P1 |
| **低危内容** | P3 | P2 | P2 |
| **正面/中性** | P3 | P3 | P3 |

### 发展态势修正

| 态势 | 定义 | 修正 |
|------|------|------|
| 加速传播 | 转发/评论增速显著高于均值，出现变体内容 | 评级**上调一级**（P0 不变） |
| 稳定 | 正常传播节奏 | 不修正 |
| 衰减 | 发布>72h 且互动数据已趋平 | 评级**下调一级**（P3 不变） |

---

## 维度一：内容严重程度

### 高危内容（触发任一即为高危）
- 人身安全、产品安全事故、伤亡
- 法律诉讼、行政处罚、监管调查
- 政治敏感、民族宗教等红线话题（品牌被卷入）
- 内部信息/商业机密泄露
- 有组织的竞品攻击/黑公关（多账号协同、模板化文案）
- 未成年人安全/隐私
- 数据安全、隐私泄露

### 中危内容
- 核心产品功能缺陷/质量问题的详细描述
- 客服/售后体验的强烈投诉（附证据）
- KOL/大V 发布的负面评价
- 被权威媒体引用或转载的负面内容
- 多平台同时出现的同类负面话题
- 涉及物流/配送的集中性投诉

### 低危内容
- 一般性使用不满（无具体证据）
- 个人偏好性的负面评价
- 历史问题的零星重提
- 单一用户的情绪发泄（无传播）

## 维度二：传播影响力

| 量级 | 播放量 | 点赞 | KOL 粉丝量 | 媒体转载 |
|------|--------|------|-----------|---------|
| **高** | >100万 | >5万 | >100万 | ≥3家 |
| **中** | 1万-100万 | 500-5万 | 1万-100万 | 1-2家 |
| **低** | <1万 | <500 | 普通用户 | 0 |

**微调规则**：
- 无互动数据时按发布者类型推断：KOL→中，普通用户→低
- 互动增速快 → 影响力上调一级
- 多平台同类话题→取最高单平台影响力计

---

## P0 红线清单（满足任一即为 P0）

1. 安全事故、人身伤害
2. 法律/监管行动
3. 政治敏感、红线话题
4. 大规模传播中的品牌核心攻击（播放>100万 **且** 针对品牌/产品核心价值）
5. 内部信息泄露
6. 有组织黑公关攻击
7. KOL(>100万粉)严重负面 + 传播加速

**P0 优先原则**：识别到红线→先标 P0，再补全其余字段。

---

## 关键边界

### P0 ↔ P1
- 高危 + 高/中影响力 → P0
- 高危 + 低影响力 → P1
- **安全/合规类红线触发时不降为 P1**

### P1 ↔ P2
- 中危 + 中影响力 → P1
- 中危 + 低影响力 → P2
- **同类问题聚集**（≥3个独立报告）→ P2 升级 P1

### P2 ↔ P3
- 低危 + 中影响力 → P2
- 低危 + 低影响力 → P3
- **实质信息存在**（有具体细节→P2；纯情绪→P3）
