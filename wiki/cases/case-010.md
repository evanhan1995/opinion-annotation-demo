## 判据链

1. **内容严重程度 = 高危**：触发"数据安全、隐私泄露"红线。用户提供了 Android 权限仪表盘截图作为技术证据，且提及 GDPR/CCPA/集体诉讼，将话题从一般抱怨升级到法律合规维度
2. **传播影响力 = 低**：普通用户(非KOL)、234赞(<1万)、89评论、未被媒体转载
3. **发展态势 = 稳定**：发布2天，数据已趋平，未出现加速传播
4. **严重度 = P1**：高危 × 低影响力 × 稳定

## 边界讨论

**为什么不是 P0？**
- P0 要求"安全/合规类红线触发时**不降**为 P1"，但该规则适用于"高危+高/中影响力"
- 此处影响力为低：普通用户、低互动、无媒体跟进、无监管机构介入
- 如果后续出现以下任一信号 → 升级 P0：
  - 媒体转载(≥1家)
  - 监管机构表示关注
  - 同类报告≥3个独立用户
  - 帖子被推至 r/all 首页(影响力↑)

**为什么不是 P2？**
- 一般产品质量投诉(如 case-002)属于中危，但此处涉及"数据隐私+法律合规暗示"，内容严重度升级为高危
- 对比 case-002(云台异响=P2)：功能缺陷 vs 隐私合规 = 不同内容层级的严重度
- 即使影响力相当(case-002 获847赞 vs 本案例234赞)，内容严重度差异决定了 P1 vs P2

**P1 的本质**：这是一个"预警区"级别——不需要立即拉群响应，但需要法务侧备案和主动监控。它是最容易被漏报的区间：既不像 P0 那样引人注目，又比 P2 更需要组织响应。

## 对标注规范的影响

1. **新增 P1 触发条件**：数据隐私/安全类内容 + 有可验证技术证据(截图/日志/复现步骤) + 低影响力 → P1（而非自动升级P0或降为P2）
2. **P1↔P0 升级通道**：同类报告聚集(≥3) 或 媒体/监管介入 → P1→P0。此条件应写入 PROMPT_LAYERS
3. **与 case-001 的对称性**：case-001 验证了"高危×高影响力×加速=P0"，本案例验证了同一高危内容在低影响力下的降级逻辑
4. **规范化提示**：标注时应区分"内容严重程度"(独立于传播)和"传播影响力"两个轴——隐私合规内容即使零传播，其内容严重度仍为高危
